# 要件定義書 (Requirements Definition)

## 1. プロジェクト概要
### 1.1 目的
- 開発者自身が毎日読むAI論文を効率的に蓄積・管理し、学習の定着を図る。
- 蓄積したナレッジを外部公開し、他の研究者やエンジニアの論文探索を支援する。

### 1.2 コンセプト
- **"Minimal & Fast"**: サーバーレスで動作する超軽量なデータベース。
- **"AIDB-like UI"**: 左サイドバーにフィルター、メインにカードリストを配置した、情報の視認性が高いデザイン。
- **"Searchable"**: 自然言語に近い検索体験（曖昧検索）を提供する。

## 2. ターゲットユーザー
- **Primary**: 開発者本人（管理者）。日々の論文読み込みのアウトプットとして利用。
- **Secondary**: 最新のAIトレンドを追いたいエンジニア・リサーチャー。

## 3. 機能要件 (Functional Requirements)

### 3.1 閲覧機能 (Viewer)
- **論文一覧表示**: カード形式で論文を表示する。
  - 表示項目: タイトル、発行日、タグ、要約（冒頭）、リンクボタン。
- **詳細表示**: カードをクリック、または「もっと見る」ボタンで全文要約を表示（モーダルまたはアコーディオン展開）。
- **ソート**: 新着順（日付降順）をデフォルトとする。

### 3.2 検索・フィルタリング機能 (Search & Filter)
- **キーワード検索**:
  - タイトル、要約、タグを対象とした横断検索。
  - `Fuse.js` を用いたファジー検索（多少のスペルミスを許容）。
- **タグフィルタリング**:
  - サイドバーに登録済みタグの一覧を表示。
  - タグをクリックすると、そのカテゴリの論文のみに絞り込まれる（AND/OR検索は現状ORでよい）。
  - 各タグの横に、該当する論文数を表示（例: `LLM (12)`）。

### 3.3 データ管理機能 (Data Management)
- **JSONベース管理**:
  - 管理画面（Admin UI）は作成しない。
  - `src/data/papers.json` を直接編集することでデータを追加・更新する。
- **データ構造**:
  - ID, Title, Summary, Tags, PublishedDate, URL, Authors を保持する。

## 4. 非機能要件 (Non-Functional Requirements)
- **パフォーマンス**:
  - 初回ロード時間を可能な限り短縮する。
  - 検索・フィルタリングは100ms以内に反応すること（クライアントサイド処理）。
- **インフラ・デプロイ**:
  - ホスティング: GitHub Pages。
  - デプロイフロー: GitHub Actionsにより、`main` ブランチへのプッシュ時に自動ビルド・デプロイ。
- **SEO・シェア**:
  - 各論文ページ（またはトップページ）のOGP設定を考慮し、X（旧Twitter）でのシェア時に見栄え良くする。

## 5. UI/UX デザインガイドライン
- **レイアウト**:
  - **PC**: 2カラム構成（左: 検索・フィルター / 右: 論文リスト）。
  - **Mobile**: 1カラム構成。フィルターはハンバーガーメニューまたは上部ドロワーに収納。
- **カラースキーム**:
  - ベース: 白または薄いグレー（読みやすさ重視）。
  - アクセント: 青系（知的な印象）。
  - タグ: カテゴリごとに視認しやすいバッジデザイン。

## 6. 技術スタック (Tech Stack)
- **Frontend**: React, TypeScript, Vite
- **Styling**: Tailwind CSS
- **Icons**: Lucide React
- **Search Logic**: Fuse.js
- **Data Source**: Local JSON file
- **Hosting**: GitHub Pages

## 7. 今後の拡張性 (Out of Scope for MVP)
- 論文ごとの個別URL（ルーティング）の実装。
- ユーザーによる「いいね」や「ブックマーク」機能（Local Storage利用）。
- 英語/日本語の言語切り替え。
